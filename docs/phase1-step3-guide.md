# フェーズ 1 ステップ 3: 動作確認とコミット

## 概要

フェーズ 1 のステップ 2（Vue.js を CDN 経由で導入）が完了したので、続いてステップ 3（動作確認とコミット）を進めます。

## ステップ 3: 動作確認とコミット

### 3.1 動作確認チェックリスト

以下の項目を確認してください：

#### 3.1.1 Vue.js の読み込み確認

1. **ブラウザで `operator.html` を開く**
2. **開発者ツールのコンソールを確認**
   - ✅ `[Vue Experiment] Vue.js が読み込まれました 3.3.4` というメッセージが表示される
   - ✅ `[Vue Experiment] Vue アプリがマウントされました` というメッセージが表示される
   - ✅ `[Vue Experiment] Vue アプリをマウントしました` というメッセージが表示される

#### 3.1.2 テストコンポーネントの確認

1. **画面右下にテストコンポーネントが表示される**
   - ✅ 「Vue.js 実験中」というメッセージ
   - ✅ カウントが 1 秒ごとに増える
   - ✅ Vue のバージョン（3.3.4）が表示される

#### 3.1.3 既存機能の確認

1. **既存の `EventAdminApp` が正常に動作する**

   - ✅ ログイン機能が動作する
   - ✅ イベント一覧が表示される
   - ✅ 質問表示など、既存機能が壊れていない

2. **エラーがない**
   - ✅ コンソールにエラーが表示されない
   - ✅ ネットワークタブで Vue.js が正常に読み込まれている

### 3.2 問題が発生した場合

#### Vue.js が読み込まれない場合

1. **ネットワーク接続を確認**

   - CDN へのアクセスが可能か確認

2. **CDN の URL を確認**

   - `https://unpkg.com/vue@3.3.4/dist/vue.global.js` が正しいか確認

3. **ブラウザのコンソールでエラーを確認**
   - ネットワークエラーがないか確認

#### 既存機能が壊れた場合

1. **Vue.js のスクリプトを一時的にコメントアウト**

   ```html
   <!-- <script src="https://unpkg.com/vue@3.3.4/dist/vue.global.js"></script> -->
   ```

2. **既存機能が復旧するか確認**

   - 復旧する場合: Vue.js との競合が原因
   - 復旧しない場合: 他の原因を調査

3. **段階的に追加して問題箇所を特定**
   - Vue.js のスクリプトだけ追加
   - テストコンポーネントを追加
   - 問題が発生する箇所を特定

### 3.3 コミット

動作確認が完了したら、変更をコミットします。

#### ステップ 3-1: 変更を確認

```bash
# 現在のブランチを確認
git branch

# 変更内容を確認
git status

# 変更の差分を確認
git diff operator.html
```

#### ステップ 3-2: 変更をステージング

```bash
# operator.html をステージング
git add operator.html
```

#### ステップ 3-3: コミット

```bash
# コミットメッセージを付けてコミット
git commit -m "feat: Vue.js を CDN 経由で導入（実験）

- operator.html に Vue.js 3.3.4 を CDN から追加
- 動作確認用のテストコンポーネントを追加
- 既存コードとの共存を確認"
```

#### ステップ 3-4: GitHub にプッシュ

```bash
# 実験ブランチを GitHub にプッシュ
git push origin experiment/vue-cdn
```

### 3.4 コミットメッセージの例

**良い例:**

```
feat: Vue.js を CDN 経由で導入（実験）

- operator.html に Vue.js 3.3.4 を CDN から追加
- 動作確認用のテストコンポーネントを追加
- 既存コードとの共存を確認

フェーズ 1 ステップ 2 完了
```

**悪い例:**

```
vue追加
```
